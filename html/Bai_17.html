Bài 17. Euler 1D solver: Godunov, MUSCL, WENO5, Runge-Kutta 3¶

Giống như bài 13, 14 ta sẽ thêm vào chương trình các sơ đồ MUSCL, WENO5. Điểm khác biệt giữa sơ đồ Godunov, MUSCL, WENO5 mà ta sử dụng nằm ở hàm tái cấu trúc nghiệm, còn các phần khác đều giống nhau và đã được lưu trong module euler_solver từ bài 16.

Đối với sơ đồ MUSCL ta chỉ xét giới hạn độ dốc minmod, đối với WENO5 ta xét trường hợp WENO5-Reconstruction. Các trường hợp còn lại các bạn tự nghiên cứu.

Code Phần code module euler_solver có chút thay đổi so với bài 16: thêm vào sơ đồ phương pháp Runge-Kutta bậc 1, bậc 3 như ta đã làm ở bài 14. Các bạn xem trong file euler_solver.py.

#sơ đồ MUSCL 
def muscl_reconstr(Ps):
    P_left = Ps[:-1].copy()
    P_right = Ps[1:].copy()
    rs = (Ps[1:-1] - Ps[:-2])/(Ps[2:] - Ps[1:-1] + 1e-10)
    
    P_left[1:] = Ps[1:-1] + 0.5*minmod(rs[:])*(Ps[2:] - Ps[1:-1])
    P_left[0] = Ps[0] #điều kiện biên 
    
    P_right[:-1] = Ps[1:-1] - 0.5*minmod(rs[:])*(Ps[2:] - Ps[1:-1])
    P_right[-1] = Ps[-1] #điều kiện biên
    return P_left, P_right

#P=(r, u, p): amax(a, P) = (max(a, r), max(a, u), max(a, p))
def amax(a, P):
    return np.array([max(a, pi) for pi in P])

def amin(a, P):
    return np.array([min(a, pi) for pi in P])

def minmod(rs):
    phi = np.array([amax(0, amin(1, r)) for r in rs])
    return phi

#sơ đồ WENO5
def psum(a, P3):
    return a[0]*P3[0] + a[1]*P3[1] + a[2]*P3[2]

#Hàm tái cấu trúc nghiệm
def reconstr(Ps, beta, a_b, gamma, a_P, Ps_05):
    for i in range(2, len(Ps)-2): #for each cell in cells
        #bước 1. tính chỉ thị độ trơn trong từng khuôn S_j
        b0 = beta(a_b[0], Ps[i-2:i+1])
        b1 = beta(a_b[1], Ps[i-1:i+2])
        b2 = beta(a_b[2], Ps[i  :i+3])
        
        #bước 2. tìm trọng lượng phi tuyến từng khuôn
        omg_r = omega(gamma, np.array([b0[0], b1[0], b2[0]]))
        omg_u = omega(gamma, np.array([b0[1], b1[1], b2[1]]))
        omg_p = omega(gamma, np.array([b0[2], b1[2], b2[2]]))
    
        #bước 3. tìm xấp xỉ bậc 3
        P0 = psum(a_P[0], Ps[i-2:i+1])
        P1 = psum(a_P[1], Ps[i-1:i+2])
        P2 = psum(a_P[2], Ps[i  :i+3])
        
        #bước 4. tìm xấp xỉ bậc 5
        Ps_05[i, 0] = sum(omg_r*[P0[0], P1[0], P2[0]])
        Ps_05[i, 1] = sum(omg_u*[P0[1], P1[1], P2[1]])
        Ps_05[i, 2] = sum(omg_p*[P0[2], P1[2], P2[2]])
        
        #Điều kiện biên: u_05[0], u_05[1], u_05[-2], u_05[-1]
    return

def weno5_reconstr(Ps):
    P_left = Ps.copy()
    P_right = Ps.copy()
    reconstr(Ps, beta_Rct, a_bRct, gamma_Rct_p05, a_uRct_p05, P_left)
    reconstr(Ps, beta_Rct, a_bRct, gamma_Rct_m05, a_uRct_m05, P_right)
    
    return P_left[:-1], P_right[1:]

Đánh giá kết quả¶

Những bài toán mà chúng ta dùng để so sánh đều có đặc trưng chung đó là có nhiều gián đoạn, đứt gãy. Kết quả cho ta thấy phương pháp tái cấu trúc hằng số từng mảnh Godunov cho kết quả kém chính nhất. Tái cấu trúc MUSCL và WENO5 cho kết quả khá tương đồng, tuy nhiên WENO5 làm xuất hiện nhiều dao động. Đối với các dạng bài toán có tính chất như trên ta nên sử dụng tái cấu trúc MUSCL.

Như ở bài 15 đã đánh giá, WENO5 chiếm ưu thế so với MUSCL khi trong bài toán có nhiều điểm cực trị. Ta sẽ kiểm chứng bằng bài toán Shu-Osher -(tham khảo).

Kết quả cho thấy, sơ đồ WENO5 cho kết quả tốt hơn ở vùng phía sau giáp với mặt sóng xung kích. Ở vùng bắt đầu nhiễu động, để ý kĩ các bạn sẽ thấy sơ đồ WENO5 làm xuất hiện các dao động nhỏ cục bộ. Tích phân thời gian bằng phương pháp Runge-Kutta bậc 3 là chưa đủ. Các bạn hãy thử với Runge-Kutta bậc 4. Nếu sử dụng Runge-Kutta bậc 1 cho sơ đồ MUSCL sẽ thu được tình trạng tương tự, hãy kiểm tra với Runge-Kutta bậc 2.

Về mặt thời gian tính toán, sơ đồ WENO5 chạy chậm hơn. Ta cũng thấy rằng, chỉ một bài toán nhỏ như vậy, lưới 1001 điểm, sơ đồ MUSCL chạy mất 2'32". Chúng ta bắt đầu thấm thía sự chờ đợi. Cũng chính vì vậy nên cần thiết phải nghiên cứu sơ đồ ẩn và tính toán song song. Ngoài ra hãy nghiên cứu thêm phương pháp tìm áp suất P xấp xỉ ví dụ như phương pháp xấp xỉ Roe.
